= Agregar recursos a los complementos compatibles con NetApp
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Debes agregar los recursos que deseas respaldar o clonar.  Dependiendo de su entorno, los recursos pueden ser instancias de bases de datos o colecciones que desea respaldar o clonar.

.Antes de empezar
* Debe haber completado tareas como instalar el servidor SnapCenter , agregar hosts, crear conexiones del sistema de almacenamiento y agregar credenciales.
* Debes haber cargado los complementos en SnapCenter Server.


.Pasos
. En el panel de navegación izquierdo, seleccione *Recursos* y luego seleccione el complemento apropiado de la lista.
. En la página Recursos, seleccione *Agregar recurso*.
. En la página Proporcionar detalles de recursos, realice las siguientes acciones:
+
|===
| Para este campo... | Haz esto... 


 a| 
Nombre
 a| 
Introduzca el nombre del recurso.



 a| 
Host name
 a| 
Seleccione el host.



 a| 
Tipo
 a| 
Seleccione el tipo. El tipo lo define el usuario según el archivo de descripción del complemento.  Por ejemplo, base de datos e instancia.

En caso de que el tipo seleccionado tenga un padre, ingrese los detalles del padre.  Por ejemplo, si el tipo es Base de datos y el padre es Instancia, ingrese los detalles de la Instancia.



 a| 
Nombre de la credencial
 a| 
Seleccione Credencial o cree una nueva credencial.



 a| 
Senderos del monte
 a| 
Introduzca las rutas de montaje donde se monta el recurso.  Esto solo se aplica a un host de Windows.

|===
. En la página Proporcionar espacio de almacenamiento, seleccione un sistema de almacenamiento y elija uno o más volúmenes, LUN y qtrees, y luego seleccione *Guardar*.
+
Opcional: Seleccione elimage:../media/add_policy_from_resourcegroup.gif["Icono más"] Icono para agregar más volúmenes, LUN y qtrees de otros sistemas de almacenamiento.

+

NOTE: Los complementos compatibles con NetApp no ​​admiten la detección automática de recursos.  Los detalles de almacenamiento de los entornos físicos y virtuales tampoco se descubren automáticamente.  Debe proporcionar la información de almacenamiento para los entornos físicos y virtuales al crear los recursos.

+
image::../media/storage_footprint.png[Huella de almacenamiento]

. En la página Configuración de recursos, proporcione pares clave-valor personalizados para el recurso.
+

NOTE: Asegúrese de que el nombre de las claves personalizadas esté en mayúsculas.

+
image::../media/resource_settings.gif[configuración de recursos]

+
Para conocer los parámetros del complemento correspondiente, consultelink:add_resources_to_netapp_supported_plugins.html#parameters-to-configure-the-resource["Parámetros para configurar el recurso"]

. Revise el resumen y luego seleccione *Finalizar*.


.Resultado
Los recursos se muestran junto con información como el tipo, el nombre del host o clúster, los grupos de recursos y políticas asociados y el estado general.


IMPORTANT: Debe actualizar los recursos si se cambia el nombre de las bases de datos fuera de SnapCenter.

.Después de terminar
Si desea proporcionar acceso a los activos a otros usuarios, el administrador de SnapCenter debe asignar activos a esos usuarios.  Esto permite a los usuarios realizar las acciones para las que tienen permiso en los activos que les están asignados.

Después de agregar los recursos, puede modificar los detalles del recurso.  Si un recurso de complementos compatibles con NetApp tiene copias de seguridad asociadas, no se pueden modificar los siguientes campos: nombre del recurso, tipo de recurso y nombre del host.



== Parámetros para configurar el recurso

Si agrega los complementos manualmente, puede usar los siguientes parámetros para configurar el recurso en la página Configuración de recursos.



=== Complemento para MongoDB

Configuración de recursos:

* MONGODB_APP_SERVER=(para el tipo de recurso como clúster fragmentado) o MONGODB_REPLICASET_SERVER=(para el tipo de recurso como conjunto de réplicas)
* OPLOG_PATH=(Parámetro opcional en caso de que se proporcione desde el archivo MongoDB.properties)
* MONGODB_AUTHENTICATION_TYPE= (PLAIN para autenticación LDAP y None para otras)


Debe proporcionar los siguientes parámetros en el archivo MongoDB.properties:

* DESACTIVAR_INICIO_DE_PARADA_SERVICIOS=
+
** N si los servicios de inicio/parada los realiza el complemento.
** Y si los servicios de inicio/**detención los realiza el usuario.
** El parámetro opcional cuyo valor predeterminado se establece en N.


* OPLOG_PATH_= (Parámetro opcional en caso de que ya se proporcione como par clave-valor personalizado en SnapCenter).




=== Complemento para MaxDB

Configuración de recursos:

* XUSER_ENABLE (Y|N) habilita o deshabilita el uso de un xuser para MaxDB de modo que no se requiera una contraseña para el usuario de la base de datos.
* HANDLE_LOGWRITER (Y|N) ejecuta operaciones de suspensión del escritor de registro (N) o de reanudación del escritor de registro (Y).
* DBMCLICMD (path_to_dbmcli_cmd) especifica la ruta al comando dbmcli de MaxDB.  Si no se configura, se utiliza dbmcli en la ruta de búsqueda.



NOTE: Para el entorno de Windows, la ruta debe estar entre comillas dobles ("...").

* SQLCLICMD (path_to_sqlcli_cmd) especifica la ruta al comando sqlcli de MaxDB.  Si no se establece la ruta, se utiliza sqlcli en la ruta de búsqueda.
* MAXDB_UPDATE_HIST_LOG (Y|N) indica al programa de respaldo de MaxDB si debe actualizar el registro del historial de MaxDB.
* MAXDB_CHECK_SNAPSHOT_DIR: Por ejemplo, SID1:directorio[,directorio...]; [SID2:directorio[,directorio...] verifica que una operación de copia de instantánea de Snap Creator sea exitosa y garantiza que se cree la instantánea.
+
Esto se aplica únicamente a NFS.  El directorio debe apuntar a la ubicación que contiene el directorio .snapshot.  Se pueden incluir varios directorios en una lista separada por comas.

+
En MaxDB 7.8 y versiones posteriores, la solicitud de copia de seguridad de la base de datos se marca como Fallida en el historial de copias de seguridad.

* MAXDB_BACKUP_TEMPLATES: especifica una plantilla de respaldo para cada base de datos.
+
La plantilla debe existir y ser un tipo de plantilla de respaldo externa.  Para habilitar la integración de instantáneas para MaxDB 7.8 y versiones posteriores, debe tener la funcionalidad de servidor en segundo plano MaxDB y una plantilla de respaldo MaxDB ya configurada del tipo EXTERNO.

* MAXDB_BG_SERVER_PREFIX: especifica el prefijo para el nombre del servidor de fondo.
+
Si se configura el parámetro MAXDB_BACKUP_TEMPLATES, también debe configurar el parámetro MAXDB_BG_SERVER_PREFIX.  Si no establece el prefijo, se utiliza el valor predeterminado na_bg_.





=== Complemento para SAP ASE

Configuración de recursos:

* SYBASE_SERVER (data_server_name) especifica el nombre del servidor de datos Sybase (opción -S en el comando isql).  Por ejemplo, p_test.
* SYBASE_DATABASES_EXCLUDE (db_name) permite excluir bases de datos si se utiliza la construcción "ALL".
+
Puede especificar varias bases de datos utilizando una lista separada por punto y coma.  Por ejemplo: pubs2;test_db1.

* SYBASE_USER: user_name especifica el usuario del sistema operativo que puede ejecutar el comando isql.
+
Requerido para UNIX.  Este parámetro es necesario si el usuario que ejecuta los comandos de inicio y detención del Agente Snap Creator (normalmente el usuario root) y el usuario que ejecuta el comando isql son diferentes.

* SYBASE_TRAN_DUMP db_name:directory_path le permite realizar un volcado de transacciones de Sybase después de crear una instantánea.  Por ejemplo, pubs2:/sybasedumps/ pubs2
+
Debe especificar cada base de datos que requiera un volcado de transacción.

* SYBASE_TRAN_DUMP_COMPRESS (Y|N) habilita o deshabilita la compresión de volcado de transacciones nativa de Sybase.
* SYBASE_ISQL_CMD (por ejemplo, /opt/sybase/OCS-15_0/bin/isql) define la ruta al comando isql.
* SYBASE_EXCLUDE_TEMPDB (Y|N) le permite excluir automáticamente las bases de datos temporales creadas por el usuario.




=== Complemento para aplicaciones Oracle (ORASCPM)

Configuración de recursos:

* SQLPLUS_CMD especifica la ruta a SQLplus.
* ORACLE_DATABASES enumera las bases de datos Oracle que se respaldarán y el usuario correspondiente (database:usuario).
* CNTL_FILE_BACKUP_DIR especifica el directorio para la copia de seguridad del archivo de control.
* ORA_TEMP especifica el directorio para archivos temporales.
* ORACLE_HOME especifica el directorio donde está instalado el software de Oracle.
* ARCHIVE_LOG_ONLY especifica si se debe realizar una copia de seguridad de los registros de archivo o no.
* ORACLE_BACKUPMODE especifica si se debe realizar una copia de seguridad en línea o fuera de línea.
* ORACLE_EXPORT_PARAMETERS especifica si las variables de entorno definidas anteriormente deben volver a exportarse mientras se ejecuta _/bin/su <user running sqlplus> -c sqlplus /nolog <cmd>_.  Este suele ser el caso cuando el usuario que ejecuta sqlplus no ha configurado todas las variables de entorno necesarias para conectarse a la base de datos mediante _connect / as sysdba_.
