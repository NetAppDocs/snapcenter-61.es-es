= Agregar recursos manualmente al host del complemento
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
El descubrimiento automático no es compatible con ciertas instancias de HANA.  Debes agregar estos recursos manualmente.

.Antes de empezar
* Debe haber completado tareas como instalar el servidor SnapCenter , agregar hosts, configurar conexiones del sistema de almacenamiento y agregar la clave de almacenamiento de usuario de HDB.
* Para la replicación del sistema SAP HANA, se recomienda agregar todos los recursos de ese sistema HANA en un grupo de recursos y realizar una copia de seguridad del grupo de recursos.  Esto garantiza una copia de seguridad sin problemas durante el modo de recuperación y toma de control.


link:task_create_resource_groups_and_attach_policies.html["Crear grupos de recursos y adjuntar políticas"] .

.Acerca de esta tarea
El descubrimiento automático no es compatible con las siguientes configuraciones:

* Diseños RDM y VMDK
+

NOTE: En caso de que se descubran los recursos mencionados anteriormente, las operaciones de protección de datos no se respaldarán en estos recursos.

* Configuración de múltiples hosts de HANA
* Varias instancias en el mismo host
* Replicación del sistema HANA de escalamiento horizontal de múltiples niveles
* Entorno de replicación en cascada en modo de replicación del sistema


.Pasos
. En el panel de navegación izquierdo, seleccione el complemento SnapCenter para la base de datos SAP HANA de la lista desplegable y luego haga clic en *Recursos*.
. En la página Recursos, haga clic en *Agregar base de datos SAP HANA*.
. En la página Proporcionar detalles de recursos, realice las siguientes acciones:
+
|===
| Para este campo... | Haz esto... 


 a| 
Tipo de recurso
 a| 
Introduzca el tipo de recurso.  Los tipos de recursos son contenedor único, contenedor de base de datos multiinquilino (MDC) y volumen sin datos.



 a| 
Nombre del sistema HANA
 a| 
Introduzca el nombre descriptivo del sistema SAP HANA.  Esta opción solo está disponible si seleccionó los tipos de recursos Contenedor único o MDC.



 a| 
SID
 a| 
Introduzca el ID del sistema (SID).  El sistema SAP HANA instalado se identifica mediante un único SID.



 a| 
Host de complementos
 a| 
Seleccione el host del complemento.



 a| 
Claves de almacenamiento seguro de usuarios de HDB
 a| 
Introduzca la clave para conectarse al sistema SAP HANA.

La clave contiene la información de inicio de sesión para conectarse a la base de datos.

Para la replicación del sistema SAP HANA, la clave de usuario secundaria no está validada.  Esto se utilizará durante la adquisición.



 a| 
Usuario del sistema operativo HDBSQL
 a| 
Introduzca el nombre de usuario para el cual está configurada la clave de almacenamiento de usuario seguro de HDB.  Para Windows, es obligatorio que el usuario del sistema operativo HDBSQL sea el usuario del SISTEMA.  Por lo tanto, debe configurar la clave de almacenamiento de usuario seguro HDB para el usuario del SISTEMA.

|===
. En la página Proporcionar espacio de almacenamiento, seleccione un sistema de almacenamiento y elija uno o más volúmenes, LUN y qtrees, y luego haga clic en *Guardar*.
+
Opcional: Puede hacer clic en el *image:../media/add_policy_from_resourcegroup.gif["Agregar formulario de política al grupo de recursos"] * Icono para agregar más volúmenes, LUN y qtrees de otros sistemas de almacenamiento.

. Revise el resumen y luego haga clic en *Finalizar*.
+
Las bases de datos se muestran junto con información como el SID, el host del complemento, los grupos de recursos y políticas asociados, y el estado general.



Si desea proporcionar a los usuarios acceso a los recursos, debe asignar los recursos a los usuarios.  Esto permite a los usuarios realizar las acciones para las que tienen permiso en los activos que les están asignados.

link:https://docs.netapp.com/us-en/snapcenter/install/task_add_a_user_or_group_and_assign_role_and_assets.html["Agregar un usuario o grupo y asignarle roles y activos"]

Después de agregar las bases de datos, puede modificar los detalles de la base de datos de SAP HANA.

No puede modificar lo siguiente si hay copias de seguridad asociadas con el recurso SAP HANA:

* Contenedores de bases de datos multiinquilino (MDC): SID o cliente HDBSQL (complemento) Host
* Contenedor único: SID o host de cliente HDBSQL (complemento)
* Volumen sin datos: nombre del recurso, SID asociado o host del complemento
