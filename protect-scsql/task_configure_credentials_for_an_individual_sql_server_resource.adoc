= Configurar credenciales para un recurso individual de SQL Server
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Puede configurar credenciales para realizar trabajos de protección de datos en recursos individuales de SQL Server para cada usuario.  Si bien puede configurar las credenciales de forma global, es posible que desee hacerlo solo para un recurso en particular.

.Acerca de esta tarea
* Si utiliza credenciales de Windows para la autenticación, debe configurar sus credenciales antes de instalar complementos.
+
Sin embargo, si está utilizando una instancia de SQL Server para la autenticación, debe agregar la credencial después de instalar los complementos.

* Si ha habilitado la autenticación SQL al configurar las credenciales, la instancia o base de datos detectada se muestra con un ícono de candado de color rojo.
+
Si aparece el ícono del candado, debe especificar las credenciales de la instancia o base de datos para agregar correctamente la instancia o base de datos a un grupo de recursos.

* Debe asignar la credencial a un usuario de control de acceso basado en roles (RBAC) sin acceso de administrador de sistemas cuando se cumplan las siguientes condiciones:
+
** La credencial se asigna a una instancia de SQL.
** La instancia o host de SQL se asigna a un usuario RBAC.
+
El usuario debe tener privilegios tanto de grupo de recursos como de respaldo.







== Paso 1: Agregar y configurar credenciales

. En el panel de navegación izquierdo, seleccione *Configuración*.
. En la página de Configuración, seleccione *Credencial*.
+
.. Para agregar una nueva credencial, seleccione *Nuevo*.
.. En la página Credencial, configure las credenciales:
+
[cols="1,3"]
|===
| Para este campo... | Haz esto... 


 a| 
Nombre de la credencial
 a| 
Introduzca un nombre para las credenciales.



 a| 
Nombre de usuario
 a| 
Introduzca el nombre de usuario utilizado para la autenticación de SQL Server.

*** Administrador de dominio o cualquier miembro del grupo de administradores. Especifique el administrador de dominio o cualquier miembro del grupo de administradores en el sistema donde va a instalar el complemento de SnapCenter . Los formatos válidos para el campo *Nombre de usuario* son:
+
**** _NetBIOS\Nombre de usuario_
**** _Dominio FQDN\Nombre de usuario_


*** Administrador local (solo para grupos de trabajo). Para los sistemas que pertenecen a un grupo de trabajo, especifique el administrador local integrado en el sistema donde va a instalar el complemento de SnapCenter . Puede especificar una cuenta de usuario local que pertenezca al grupo de administradores locales si esta cuenta tiene privilegios elevados o si la función de control de acceso de usuario está deshabilitada en el sistema host.  El formato válido para el campo *Nombre de usuario* es: _Nombre de usuario_




 a| 
Password
 a| 
Introduzca la contraseña utilizada para la autenticación.



 a| 
Modo de autenticación
 a| 
Seleccione el modo de autenticación de SQL Server.  También puede elegir la autenticación de Windows si el usuario de Windows tiene privilegios de administrador de sistemas en el servidor SQL.



 a| 
Host
 a| 
Seleccione el host.



 a| 
Instancia de SQL Server
 a| 
Seleccione la instancia de SQL Server.

|===
.. Seleccione *Aceptar* para agregar la credencial.






== Paso 2: Configurar instancias

. En el panel de navegación izquierdo, seleccione *Recursos*.
. En la página Recursos, seleccione *Instancia* de la lista *Ver*.
+
.. Seleccione image::../media/filter_icon.png[icono de filtro] y luego seleccione el nombre del host para filtrar las instancias.
.. Seleccione image::../media/filter_icon.png[icono de filtro] para cerrar el panel de filtro.


. En la página Proteger instancia, proteja la instancia y, si es necesario, seleccione *Configurar credenciales*.
+
Si el usuario que ha iniciado sesión en SnapCenter Server no tiene acceso al complemento SnapCenter para Microsoft SQL Server, entonces el usuario debe configurar las credenciales.

+

NOTE: La opción de credenciales no se aplica a bases de datos y grupos de disponibilidad.

. Seleccione *Actualizar recursos*.
