= Realizar copias de seguridad de los grupos de recursos de bases de datos de Oracle
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Un grupo de recursos es una colección de recursos en un host o clúster.  La operación de copia de seguridad se realiza en todos los recursos definidos en el grupo de recursos.

Puede realizar una copia de seguridad de un grupo de recursos a pedido desde la página Recursos.  Si un grupo de recursos tiene una política adjunta y una programación configurada, las copias de seguridad se crean de acuerdo con la programación.

.Pasos
. En el panel de navegación izquierdo, seleccione *Recursos* y el complemento apropiado de la lista.
. En la página Recursos, seleccione *Grupo de recursos* de la lista *Ver*.
. Ingrese el nombre del grupo de recursos en el cuadro de búsqueda o haga clic enimage:../media/filter_icon.gif["icono de filtro"] y seleccione la etiqueta.
+
Hacer clicimage:../media/filter_icon.gif["icono de filtro"] para cerrar el panel de filtro.

. En la página Grupo de recursos, seleccione el grupo de recursos que desea respaldar.
+

NOTE: Si tiene un grupo de recursos federado con dos bases de datos y una tiene datos en un almacenamiento que no es de NetApp , la operación de respaldo se cancela incluso si la otra base de datos está en un almacenamiento de NetApp .

. En la página Copia de seguridad, realice los siguientes pasos:
+
.. Si tiene varias políticas asociadas con el grupo de recursos, seleccione la política de respaldo que desea utilizar en la lista desplegable *Política*.
+
Si la política seleccionada para la copia de seguridad a pedido está asociada con una programación de copia de seguridad, las copias de seguridad a pedido se conservarán según la configuración de retención especificada para el tipo de programación.

.. Seleccione *Copia de seguridad*.


. Supervise el progreso seleccionando *Monitor > Trabajos*.


.Después de terminar
* En la configuración de AIX, puede utilizar el `lkdev` comando para bloquear y el `rendev` comando para cambiar el nombre de los discos en los que residía la base de datos de la que se hizo una copia de seguridad.
+
Bloquear o cambiar el nombre de los dispositivos no afectará la operación de restauración cuando restaure utilizando esa copia de seguridad.

* Si la operación de copia de seguridad falla porque el tiempo de ejecución de la consulta de la base de datos excedió el valor de tiempo de espera, debe cambiar el valor de los parámetros ORACLE_SQL_QUERY_TIMEOUT y ORACLE_PLUGIN_SQL_QUERY_TIMEOUT ejecutando el comando `Set-SmConfigSettings` cmdlet:
+
Después de modificar el valor de los parámetros, reinicie el servicio SnapCenter Plug-in Loader (SPL) ejecutando el siguiente comando `/opt/NetApp/snapcenter/spl/bin/spl restart`

* Si el archivo no es accesible y el punto de montaje no está disponible durante el proceso de verificación, la operación podría fallar con el código de error DBV-00100 del archivo especificado.  Debe modificar los valores de los parámetros VERIFICATION_DELAY_ y VERIFICATION_RETRY_COUNT en sco.properties.
+
Después de modificar el valor de los parámetros, reinicie el servicio SnapCenter Plug-in Loader (SPL) ejecutando el siguiente comando `/opt/NetApp/snapcenter/spl/bin/spl restart`
